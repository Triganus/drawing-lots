# Жребий — сервис принятия вдохновляющих решений

Одностраничное приложение на React, которое помогает «позвать судьбу» и выбрать направление отпуска или любой другой вариант: пользователь вводит вопрос, варианты, а сайт красиво подсвечивает победителя, добавляет аргумент «за» и моковые подборки билетов и отелей.

## Быстрый старт

1. Склонируйте или скачайте репозиторий.
2. Откройте `index.html` в браузере (двойной клик или через локальный http-сервер).
3. Введите вопрос и варианты — минимум два. Жмите «Тянуть жребий».

## Стек

- **React 18** через CDN (UMD) + Babel Standalone для JSX в браузере.
- **HTML/CSS** без сборщиков — всё работает из коробки.
- **Unsplash** (через прямые ссылки) для фоновых фото.

## Возможности

- Красивое окно с вопросом, вариантами и анимацией выбора.
- Моковые аргументы «за» и истории крайних решений.
- Автоматический подбор тематических фоновых фотографий.
- Дополнительный блок «Подборка для поездки» с условными предложениями билетов и отелей.
- Адаптивная вёрстка для планшетов и телефонов.

## Структура

```
├── index.html   # Точка входа, подключает стили и React
├── styles.css   # Оформление: градиенты, карточки, адаптив
└── app.js       # Логика приложения и моковые данные
```

## Как подружить с реальными сервисами

- Подключить виджеты Aviasales, Booking, TripAdvisor или других партнёров через их affiliate-программы.
- Вынести работу с API на бэкенд (Node/Express) для защиты ключей.
- Вместо моковых данных использовать реальные ответы и отображать результаты в карточках.

## Идеи для развития

- Сохранение историй в localStorage или бэкенд.
- Шеринг результата в соцсети.
- Подбор маршрутов, плейлистов, чек-листов под выбранную страну.
- Локализации на другие языки.
